Why Password Protect a PDF?

A password-protected PDF is encrypted — anyone who opens it without the correct password sees only a prompt asking for one. No content is visible, no text is selectable, no pages can be printed. This makes PDF encryption one of the most reliable ways to protect document contents during sharing.

Common situations where PDF passwords are essential: sending salary slips or tax documents by email, sharing legal agreements before signing, distributing proprietary research or reports, protecting invoices with bank details, and sending medical or HR records.

⚠️ Password Protection ≠ Digital Signature

A PDF password prevents unauthorized viewing. It does not verify the document's author or guarantee it hasn't been altered. For that, you need a digital signature. If both security and authenticity matter, use both.

Two Types of PDF Passwords

PDF security has two distinct password layers — most free tools only set the first one:

  • Open Password (User Password): Required just to open and view the file. This is what most people mean by "password protecting" a PDF. Set this when you want to restrict who can read the document.
  • Permissions Password (Owner Password): Controls what an authorized viewer can do — print, copy text, fill forms, modify pages. The document opens normally, but certain actions are locked. Useful for distributing finalized contracts or read-only reports.

For most everyday needs, an open password is sufficient. The methods below all set an open password unless otherwise noted.

🔒 Privacy Note for Online Tools

When using online PDF tools, your file is uploaded to a third-party server temporarily. For highly sensitive documents (legal, medical, financial), prefer an offline method — use the Mac Preview method, LibreOffice on Windows, or a local app on mobile.

Method 2 — Mac (Built-In, Zero Upload)

Macs have PDF encryption built directly into the system — no additional tools, no upload, completely private. This is the best method for Mac users handling sensitive documents.

  1. Open the PDF in Preview (double-click it, or right-click → Open With → Preview)
  2. Go to File → Export as PDF
  3. Click the Show Details button (bottom-left of the save dialog)
  4. Check the box: Encrypt
  5. Enter a password and verify it
  6. Click Save

The saved PDF is now encrypted with AES-128. The original unencrypted file remains untouched — Preview saves an encrypted copy. Your document never left your Mac.

✅ Also Works for Print to PDF

On Mac, if you're creating a PDF from another app (Pages, Numbers, Safari), you can encrypt it at the print step: File → Print → PDF (bottom-left dropdown) → Save as PDF → Encrypt. You get a password-protected PDF directly, skipping the intermediate unprotected file entirely.

Method 3 — Windows (LibreOffice — Free)

Windows doesn't have a built-in PDF encryption tool, but LibreOffice (free, open-source) handles it cleanly:

  1. Download and install LibreOffice from libreoffice.org (free, ~300MB)
  2. Open your PDF in LibreOffice Draw (right-click the PDF → Open With → LibreOffice Draw)
  3. Go to File → Export as PDF
  4. Click the Security tab in the PDF Options dialog
  5. Set an Open password and/or a Permissions password
  6. Click Export — save the new encrypted file

LibreOffice supports AES-256 encryption and both password types, making it the most capable free option on Windows. The trade-off is that it may slightly reformat complex PDFs when opening them in Draw.

Adobe Acrobat Reader on Windows — Limited

The free version of Adobe Acrobat Reader can open protected PDFs but cannot create them. Encrypting a PDF requires Acrobat Pro ($23.99/month). Stick with LibreOffice or an online tool for free encryption on Windows.

Method 4 — iPhone & Android

iPhone / iPad

iOS doesn't have a built-in PDF encryption tool. The easiest free option is the PDF Expert app (free tier supports password protection): open the PDF → tap the "..." menu → Protect → Set Password. Alternatively, use a browser-based tool like PDF24 in Safari — it works fully on iPhone.

Android

Xodo PDF Reader & Editor (free) supports adding passwords: open the PDF → tap the three-dot menu → Security → Set Password. Files are processed locally on your device. For a no-app option, PDF24's mobile website works well in Chrome on Android.

How to Choose a Strong PDF Password

A PDF is only as secure as its password. AES-256 encryption is essentially unbreakable by brute force — but a weak password defeats that entirely. Here's what makes a good PDF password:

  • Length matters most: A 12-character password is exponentially harder to crack than an 8-character one. Aim for at least 12 characters.
  • Mix character types: Use uppercase, lowercase, numbers, and at least one symbol (!, @, #, $, etc.).
  • Avoid predictable patterns: Dictionary words, birthdays, "password123", and keyboard walks (qwerty, 12345) are all tested first in any attack.
  • Use a passphrase: A string of 4–5 random words is both strong and memorable — "correct-horse-battery-staple" is much stronger than "P@ssw0rd".
  • Never reuse passwords: Use a unique password for each sensitive document, especially if you're distributing the same PDF to multiple people.
⚠️ Store the Password Safely

If you forget the password to an encrypted PDF, the contents are unrecoverable — that's the point of encryption. Store passwords in a password manager (Bitwarden is free and open-source), not in the filename or a sticky note on your screen.

Frequently Asked Questions

Can I remove the password from a PDF I already protected?
Yes — if you know the password. Open the PDF (enter the password), then re-export or save it without setting a password. On Mac, File → Export as PDF without checking Encrypt. In LibreOffice, export without setting a security password. Online tools like PDF24 also have a dedicated "Unlock PDF" tool.
Is a password-protected PDF truly secure?
With a strong password and AES-256 encryption, yes — the document content is mathematically unreadable without the key. The weak point is always the password itself, not the encryption. A short or common password can be cracked with freely available tools in minutes.
What's the difference between encrypting and just adding a password?
In PDF security, they're the same thing. Setting an "open password" on a PDF encrypts the entire file contents using AES. Without the password, the raw file data is unreadable ciphertext. "Encryption" and "password protection" refer to the same technical operation in the PDF specification.
